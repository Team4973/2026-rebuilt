## Project Overview

This is a RobotPy (Python WPILib) FRC robot project implementing a swerve drivetrain using CTRE Phoenix 6 hardware (TalonFX motors, CANcoders, Pigeon 2 IMU). The project uses the WPILib command-based framework.

## Build and Run Commands

This project uses `uv` for Python dependency management.

```bash
# Sync dependencies
uv sync

# Run robot simulation
uv run python -m robotpy sim

# Run tests
uv run python -m robotpy test

# Run a single test
uv run pytest tests/pyfrc_test.py

# Deploy to robot
uv run python -m robotpy deploy
```

## Architecture

### Entry Point
- `robot.py` - Main robot class (`MyRobot`) extending `TimedCommandRobot`. Initializes `RobotContainer` and handles mode transitions.

### Core Components
- `robotcontainer.py` - Configures subsystems, commands, and controller bindings. Sets up:
  - Field-centric swerve drive control
  - Xbox controller input mapping
  - SysId characterization routines
  - Simple autonomous command

### Subsystems
- `subsystems/command_swerve_drivetrain.py` - `CommandSwerveDrivetrain` class that wraps Phoenix 6's `SwerveDrivetrain` as a WPILib Subsystem. Handles:
  - Operator perspective (alliance-aware field-centric driving)
  - SysId routines for drive/steer/rotation characterization
  - Simulation support with fast update loop

### Generated Code
- `generated/tuner_constants.py` - Hardware configuration generated by CTRE Tuner X Swerve Project Generator. Contains:
  - `TunerConstants` - PID gains, gear ratios, CAN IDs, module positions, encoder offsets
  - `TunerSwerveDrivetrain` - Base class specifying TalonFX/CANcoder hardware types
  - `create_drivetrain()` - Factory method for drivetrain instantiation

### Telemetry
- `telemetry.py` - Publishes swerve state to NetworkTables and SignalLogger for debugging/tuning.

## Key Dependencies
- `robotpy[commands2,sim]~=2026.1` - WPILib and command framework
- `phoenix6~=26.1` - CTRE Phoenix 6 motor controller library

## Hardware Configuration
- 4 swerve modules (FL, FR, BL, BR) with TalonFX drive/steer motors and CANcoders
- Pigeon 2 IMU on CAN ID 0
- All devices on default CAN bus with hoot logging to `./logs/`
